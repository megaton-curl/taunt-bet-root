# Product references (non-contract)

`product-spec-v3.md` and `platform-features.md` are **product vision** — narrative, GTM, and UX intent exported from stakeholder docs. They are **not** engineering specifications: they do not define acceptance criteria, APIs, or on-chain contracts.

`challenge-engine-spec.md` is **design research / pre-spec** (competitor notes and service shape). The implementable specification is **`docs/specs/400-challenge-engine/spec.md`**.

For implementable work, use **`docs/specs/*`** (each `spec.md` with Meta, FR, and implementation checklist).
