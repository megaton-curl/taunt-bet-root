# Product references (non-contract)

`product-spec-v4.md` and `platform-features.md` are **product vision** — narrative, GTM, and UX intent exported from stakeholder docs. They are **not** engineering specifications: they do not define acceptance criteria, APIs, or on-chain contracts.

`docs/archive/references/challenge-engine-spec.md` is archived **design research / pre-spec** (competitor notes and service shape). The implementable specification is **`docs/specs/400-challenge-engine/spec.md`**.

Active `docs/references/` is intentionally text-first. Raw stakeholder binaries and superseded reference docs have been archived or removed so diffs stay reviewable.

For implementable work, use **`docs/specs/*`** (each `spec.md` with Meta, FR, and implementation checklist).
